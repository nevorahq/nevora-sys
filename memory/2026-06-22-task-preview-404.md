# Task preview 404 investigation

- Symptom: a task visible at `/dashboard/tasks` returned 404 at `/dashboard/tasks/:id`.
- Root cause: `getTaskById()` used an extended select with optional nested task relation tables. Any schema/RLS error returned `null`; the page interpreted that as a missing task and called `notFound()`.
- Fix: retain the extended query when available, then fall back to the same base `todos` access pattern used by the task list. Legacy task fields are normalised for rendering and optional detail arrays are empty.
